# Chrome Web Store 扩展程序上架指南

本指南将详细说明如何将您的 Chrome 扩展程序 (如当前的 NotebookLM 扩展) 发布到 Chrome 网上应用店 (Chrome Web Store)。

## 第一阶段：准备工作

在提交扩展程序之前，请确保完成以下准备工作：

1. **打包扩展程序目录为 ZIP 压缩包**
    - 确保包含 `manifest.json` 以及所有相关的脚本 (`content.js` 等)、样式 (`main.css`)、图标和 HTML 文件。
    - **注意：** 压缩包内不要包含 `.git` 等与扩展无关的隐藏文件夹或其他开发文档。在 macOS 上可使用以下命令排除多余文件并打包：
      ```bash
      zip -r notebooklm_ext.zip . -x "*.DS_Store" -x ".git/*" -x "*.md" 
      ```
      *(以上命令将不需要的 `.md` 说明文档等一并排除，确保提交的扩展体积精简且无无关内容)。*

2. **准备商店所需的素材**
    - **扩展程序图标**: 一张 `128x128` 像素的高分辨图标（通常已包含在 `icons/` 文件夹内部）。
    - **促销图片 (Promo Tile)**: 一张 `440x280` 像素的图片，用于商店在各种推荐位展示。如果想展示效果更好，也可以额外再准备一个 `1400x560` 大小的跑马灯推荐大图。
    - **商店屏幕截图**: 至少 1 张，至多 5 张屏幕截图，尺寸必须为 `1280x800` 像素或 `640x400` 像素。
    - **文案内容**: 提前在文档中拟好中英文的短描述和详细长描述。

## 第二阶段：注册开发者账号

如果您还没有 Chrome 开发者账号，请按以下步骤注册：

1. 浏览器访问并使用您的 Google 账号登录 [Chrome 网上应用店开发者信息中心 (Developer Dashboard)](https://chrome.google.com/webstore/devconsole/)。
2. 仔细阅读并同意《开发者协议》。
3. 支付一次性的开发者注册费 (5 美元)。您需要绑定一张支持外币付款的信用卡 (如 Visa / MasterCard)。

## 第三阶段：上传和配置扩展项目

在开发者信息中心完成注册后，可以正式开始创建并填写项目：

1. **新建项 (New Item)**:
   - 点击开发者控制台右上角的“新建项”按钮。
   - 上传您在第一阶段准备好的 `notebooklm_ext.zip` 压缩包文件。

2. **填写“商店发布内容 (Store Listing)”**:
   - **名称 (Name)**: 会自动从 `manifest.json` 中读取 (例如 NotebookLM Note Sorter)，您也可以在此页面修改以适应商店显示要求。
   - **简短描述 (Summary)**: 填写一句话描述其核心功能。
   - **详细说明 (Detailed description)**: 清晰阐述扩展程序的功能、特点以及如何使用。内容越有条理、详情越丰富，越容易通过审核，也更容易获得用户青睐。
   - **类别 (Category)**: 选择一个最契合的类别，比如“生产力工具 (Productivity)”。
   - **图形素材**: 按照页面要求，上传上文准备的图标、屏幕截图和促销图片。

3. **填写“隐私保护协议 (Privacy Practices)”** (审核重点！):
   - **单一用途 (Single Purpose)**: 必须用一两句话描述该扩展的核心用途。例如：“该扩展旨在为 NotebookLM 提供更便捷的笔记排序与界面调整功能”。
   - **权限说明 (Permission Justification)**: 如果您在 `manifest.json` 中申请了特定权限 (当前可能包含对 `notebooklm.google.com` 的 `host_permissions`)，您必须在表单里详细解释为什么需要向网页注入脚本并操作页面，且只能用于声明的核心功能。
   - **数据收集**: 如实填写扩展是否会收集用户数据。如果仅是纯本地执行的前端代码（不包含上传服务端），就明确声明**不收集任何用户数据**。

## 第四阶段：提交并等待审核

1. 依次检查左侧菜单中各项导航栏，确保所有必填项已亮起绿色的对勾提示。如果需要分发到其他国家/地区，可以进行相应的本地化设置。
2. 确认无误后，点击右上角的 **“提交审核” (Submit for Review)** 按钮。
3. **审核流程与周期**：
   - 第一次提交的扩展可能需要 1 到 7 个工作日左右的时间进行人工和机器双重审核（尤其对于申请了 `host_permissions` 注入页面脚本的扩展）。后续的常规代码迭代更新通常会在 24 小时内通过。
   - 如果遇到退回或拒绝，Chrome 开发团队会通过邮件联系您，指出具体违反了哪一条商店政策（比如权限申请过度、隐私声明模糊等）。按照邮件指出的原因进行修改，然后重新提交即可。

## 第五阶段：发布与后续更新

1. 审核一旦通过，您的扩展状态会变成“已发布 (Published)”，此时用户就可以在应用店中搜索到、查看详情并安装了。您也可以复制分享商店链接。
2. **发布更新版本**：以后如果您修改了相关的代码或是修复了 bug，只需：
   - 在 `manifest.json` 中修改 `version` 的数字（例如从 `"1.0"` 改为 `"1.1"`）
   - 重新执行命令打成 `ZIP` 包
   - 登录开发者信息中心，进入已有的该扩展，选择上传更新包覆盖
   - 重新点击“提交审核”即可
